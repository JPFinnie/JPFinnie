## What I am building toward

### Deep context systems

Structured knowledge that agents can reason over. I am interested in the shift from one-shot retrieval to persistent, maintained knowledge layers: local-first notes, entity pages, graph snapshots, semantic edges, and agent-readable operating contracts.

### Agentic product management

Product work that moves from long static documents into Prototypes, evals, and working systems. The goal is not to generate more artifacts. The goal is to make product thinking executable.

### Prototype-first delivery

I like taking an idea from "should this exist?" to a working prototype quickly, then using the prototype to sharpen judgement. Design systems, agent loops, skills files, and end-to-end pipelines are the operating layer.

### Personalisation in wealth technology

Investing software should feel like the institution understands the person using it. I am interested in timely insight, appropriate nudges, transparent tradeoffs, and decision support that respects the boundary between explanation and execution.

## Selected work

### [PRISM](https://github.com/JPFinnie/PRISM)

A deep context system for structured personal and organisational knowledge. PRISM is the working proof of the "personal LLM wiki" idea: context first, everything else built on top.

### Cerebro Atlas

A local-first Obsidian vault plus generated graph snapshot for navigating knowledge over time. The pattern combines Markdown, wikilinks, graph topology, semantic edges, and a public methodology layer so the system can be explained and rebuilt.

Public methodology: [cerebro-finnie.vercel.app/docs](https://cerebro-finnie.vercel.app/docs)

### Pulse-style financial decision engine

An investing product prototype pattern where deterministic math produces ranked actions, scenario outcomes, and risk metrics, while AI explains the result in bounded language. The important boundary: AI explains tradeoffs, but does not execute trades or invent financial judgement.

## How I work

- Start with the why, then build the smallest useful proof.
- Prefer working prototypes over abstract alignment.
- Keep deterministic logic where the stakes are high.
- Make agent context explicit, structured, and inspectable.
- Treat taste and judgement as product infrastructure.

Some of my work is necessarily described at the pattern level because it sits in regulated financial or healthcare contexts. The public work here is intended to show the operating model: context, prototypes, decision systems, and the product judgement around them.
